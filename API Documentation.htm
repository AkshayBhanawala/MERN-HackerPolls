LogIn

Login user. This will given you an authorization JWT Token that you will require to perform user specific tasks. Token will be HTTP only cookie, as a Authorization header. Without this token you can’t access the service. After a successful login, authorization token is valid for 1 hour. If already logged in, and if token is not expired, your current authorization token will be decoded and given as a response only. Database values will not be fetched from re-login operation.

  • Curl
  • HTTP
curl -X POST -d '{
    "name": "UserName",
    "password": "UserPassword"
}' "localhost:3001/API/LogIn"
POST 3001/API/LogIn HTTP/1.1

    "name": "UserName",
    "password": "UserPassword"
}

VoteUser

Vote a user. This is an IP and/or User restricted service. If a user has already voted, they can’t vote again. If someone has voted from an IP, same IP can’t be used to vote again. In response it’ll return all the users with updated details from the server and isVoteGiven as true for current logged in user or IP.

In request you send user object with name as the name of the user the vote in being given to.

One can vote self or other maximum once*.

  • HTTP
curl -X PATCH -d '{
    "user" : {
        "name": "Admin"

}' "localhost:3001/API/VoteUser"

AddUser

Add new user. Admin only restricted service. Authorization token of an admin user is required. Allows admin to add new user in the system. Request requires, user object that include minimum name and password variables. Other details are optional such as (numSolvedChallenges, expertise). Error response is given from unauthorised user. See examples for details.

DeleteUser

Delete new user. Admin only restricted service. Authorization token of an admin user is required. Allows admin to delete user from the system. Request requires, user object that include minimum name variable. Other details are ignored from request. The name specified in the user object will be deleted from the database if it exists. User exist or not, it will return a valid response. Error response is given from unauthorised user. See examples for details.

  • HTTP

EditUser

Edit a user. This is an user restricted service. Authorization token is required. Error response is given from unauthorised user. Only logged in user can edit their details. Admin also can’t edit other’s details. And login is required to access this service. In request it accepts updates for password and/or numSolvedChallenges and/or expertise. Partial update of expertise is not supported, experise object will be as it is updated in database. If you want to add new expertise with existing expertise, include existing expertise with your new expertise along with it.

  • Curl
  • HTTP
curl -X PUT -d '{
    "user" : {
        "password": "Jane'sNewPassword",
        "numSolvedChallenges": 2,
        "expertise": [
            {
                "name": "JavaScript",
                "rating": 4
            },
            {
                "name": "Python",
                "rating": 3
            }
